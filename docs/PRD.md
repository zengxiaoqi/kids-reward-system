# 产品需求文档 (PRD) — 小孩行为积分奖惩系统

> 版本: v0.1 (草案)
> 日期: 2026-06-24
> 状态: 需求调研

---

## 1. 产品概述

### 1.1 产品定位
面向 3-12 岁儿童家庭的行为习惯养成工具，通过积分激励机制，将日常行为量化为可感知的奖励，帮助孩子建立良好的生活习惯。

### 1.2 目标用户
- **主要用户**: 家长（管理行为规则、记录积分、设置礼品）
- **次要用户**: 孩子（查看积分、选择兑换礼品）

### 1.3 核心价值
- 让好习惯变得"看得见"
- 用正向激励替代说教
- 培养孩子的目标感和自律能力

## 2. 功能需求

### 2.1 行为管理模块

#### 2.1.1 固定日常行为（系统预设）
每日循环的基础行为项，可启用/禁用、调整积分值：

| 行为项 | 默认积分 | 说明 |
|--------|----------|------|
| 🌅 早上起床洗漱 | +2 | 按时完成晨间洗漱 |
| 🥣 早餐 | +1 | 好好吃完早餐 |
| 📚 学习/作业 | +3 | 完成当日学习任务 |
| 🍱 中午吃饭 | +1 | 按时吃完午餐 |
| 🍽️ 晚上吃饭 | +1 | 按时吃完晚餐 |
| 🪥 晚上洗漱 | +2 | 按时完成晚间洗漱 |
| 😴 睡觉 | +2 | 按时上床睡觉 |

#### 2.1.2 自定义行为（家长添加）
家长可随时新增行为项：

**加分行为示例：**
- 📖 课外阅读 30 分钟 → +3
- 🏃 运动/锻炼 → +3
- 🧹 帮忙做家务 → +2
- 🎹 练习乐器 → +2
- 🤝 礼貌待人/分享 → +1
- 🎨 画画/手工 → +2
- ✅ 考试进步 → +10
- 🌟 特别表扬（由家长临时给）→ 自定义积分

**扣分行为示例：**
- 😤 发脾气/哭闹 → -2
- 📱 超时使用电子设备 → -3
- ❌ 说谎 → -5
- 🗑️ 乱扔东西 → -1
- ⏰ 迟到/拖延 → -1
- 😠 打人/骂人 → -5

#### 2.1.3 行为规则配置
- 每个行为项可设置：名称、图标、积分值（正/负）、是否每日重复、时间段限制
- 行为项可分组：学习类、生活类、品德类、运动类等
- 支持设置每日积分上限（防止刷分）
- 支持设置连续完成奖励（连续 7 天完成某行为额外奖励）

### 2.2 积分记录模块

#### 2.2.1 每日打卡
- 家长手动记录：点击行为项 → 确认完成/未完成
- 快速操作：一键打卡当日所有固定行为
- 支持补录：可以回溯修改过去几天的记录
- 支持撤销：误操作可撤回

#### 2.2.2 积分流水
- 每笔积分变动都有记录：时间、行为、积分值、备注
- 可查看每日/每周/每月积分汇总
- 积分变动通知（可选推送给孩子）

### 2.3 礼品兑换模块

#### 2.3.1 礼品管理
家长设置可兑换的礼品及所需积分：

**礼品示例：**
| 礼品 | 所需积分 | 分类 |
|------|----------|------|
| 看动画片 30 分钟 | 10 | 娱乐 |
| 选一个小零食 | 15 | 食物 |
| 去公园玩 | 30 | 外出 |
| 买一本新书 | 50 | 学习 |
| 买一个玩具（50元内） | 100 | 礼物 |
| 去游乐园 | 200 | 外出 |
| 许愿卡（任意合理愿望） | 500 | 特别 |

#### 2.3.2 兑换流程
1. 孩子选择心仪的礼品
2. 确认积分是否足够
3. 家长确认兑换 → 扣除积分
4. 记录兑换历史
5. 支持"心愿单"功能 — 积分不够时先加入心愿单

#### 2.3.3 礼品分类
- 即时奖励（看动画、吃零食）
- 短期目标（买书、小玩具）
- 长期目标（游乐园、大件礼物）
- 体验类（亲子活动、选择周末去哪）

### 2.4 数据统计模块

#### 2.4.1 仪表盘
- 当前总积分
- 今日获得积分
- 本周/本月积分趋势图
- 连续打卡天数
- 距离下一个礼品还差多少积分

#### 2.4.2 行为分析
- 各项行为完成率统计
- 最常加分/扣分的行为
- 每周/每月行为报告
- 习惯养成进度（某行为连续完成 N 天标记为"已养成"）

#### 2.4.3 成就系统
- 🏅 连续打卡 7 天 — "一周小达人"
- 🏆 连续打卡 30 天 — "月度之星"
- ⭐ 累计 100 积分 — "百分小能手"
- 🎖️ 兑换第一个礼品 — "梦想启程"
- 💎 累计 1000 积分 — "积分大王"

### 2.5 多孩管理

- 支持添加多个孩子档案
- 每个孩子独立的积分账户和行为记录
- 可为不同孩子设置不同的行为规则和礼品
- 孩子之间的积分独立，不做比较

### 2.6 家长设置

- 行为规则自定义（增删改）
- 积分值调整
- 礼品库管理
- 每日打卡提醒时间设置
- 数据导出（Excel/PDF 报告）

## 3. 非功能需求

### 3.1 用户体验
- 界面友好、色彩丰富、适合儿童审美
- 操作简单，家长 3 分钟内能上手
- 孩子能看懂积分和礼品界面
- 有趣的动画反馈（获得积分时的特效）

### 3.2 数据安全
- 用户数据加密存储
- 不收集不必要的个人信息
- 支持数据导出和删除

### 3.3 平台支持
- 优先移动端（家长用手机操作最多）
- 可选 Web 端（大屏查看统计）

## 4. 技术方案调研

### 4.1 待评估方案

#### 方案 A: Flutter 跨平台 App
- 优势：一套代码同时支持 Android/iOS，与灵犀伴学技术栈一致
- 劣势：需要后端服务

#### 方案 B: 微信小程序
- 优势：无需安装，家长使用门槛低，微信生态分享方便
- 劣势：平台限制，功能受限

#### 方案 C: Web App (PWA)
- 优势：跨平台，开发快速，可离线使用
- 劣势：推送通知受限

#### 方案 D: 纯本地 App（无后端）
- 优势：简单，无需服务器，数据在本地
- 劣势：多设备同步困难

### 4.2 后端方案
- 自建后端（Node.js / Python）
- BaaS（Firebase / Supabase）
- 无后端（本地存储 + 可选云同步）

### 4.3 需要决策的问题
1. 目标平台优先级？（Android / iOS / 小程序 / Web）
2. 是否需要多设备同步？
3. 是否需要离线使用？
4. 是否需要消息推送提醒？
5. 数据量预估（行为记录、积分历史）

## 7. 开放问题

1. 积分是否有"过期"机制？（如每月清零 vs 永久累积）
2. 是否需要"惩罚"机制（扣分）还是纯正向激励？
3. 是否需要家长审批兑换，还是孩子可自主兑换？
4. 是否需要时间窗口限制？（如"晚上 9 点前睡觉"才加分）
5. 是否需要"双倍积分日"等特殊活动？
6. 多个孩子之间是否需要协作任务？（共同完成获得额外积分）
