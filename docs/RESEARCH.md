## 1. 类似产品调研

### 1.1 国内产品

#### 小日常 (Daily)
- 平台: iOS / Android
- 特点: 简洁习惯打卡，每日记录
- 优点: UI 清新，操作简单
- 不足: 没有积分和兑换机制

#### 宝宝生活记录
- 平台: 微信小程序
- 特点: 记录宝宝日常作息
- 优点: 微信生态，无需安装
- 不足: 偏记录，缺少激励

### 1.2 国外产品

#### Homey
- 平台: iOS / Android
- 特点: 家庭任务管理 + 零花钱挂钩
- 优点: 任务-奖励闭环，多孩支持
- 不足: 英文界面，付费

#### Habitica
- 平台: Web / iOS / Android
- 特点: RPG 游戏化习惯追踪
- 优点: 游戏元素丰富，成就系统完善
- 不足: 对小孩来说太复杂

#### iRewardChart
- 平台: iOS
- 特点: 可视化奖励图表
- 优点: 直观，适合低龄儿童
- 不足: 功能单一

#### Greenlight
- 平台: iOS / Android
- 特点: 儿童理财 + 任务管理
- 优点: 积分与真实金钱挂钩
- 不足: 仅美国市场

## 2. 技术方案对比

### 2.1 前端方案

| 方案 | 开发效率 | 用户体验 | 跨平台 | 维护成本 | 适合场景 |
|------|----------|----------|--------|----------|----------|
| Flutter | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Android/iOS/Web | 低 | 多平台 App |
| 微信小程序 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 微信内 | 低 | 快速上线 |
| React Native | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | Android/iOS | 中 | 已有 RN 经验 |

| 原生开发 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 单平台 | 高 | 极致体验 |

| 方案 | 开发效率 | 可扩展性 | 运维成本 | 数据控制 |

### 2.3 推荐方案（初步）

**如果优先快速验证:**
→ 微信小程序 + 云开发（无需自建后端）

**如果优先长期发展:**
→ Flutter App + Node.js 后端（与灵犀伴学技术栈一致）

**如果优先最简实现:**
→ Flutter App + 本地 Hive 存储（无后端，数据在设备本地）

## 3. 关键技术点

### 3.1 数据存储
- 行为记录：结构化数据，需支持时间范围查询
- 积分流水：只追加不修改的日志型数据
- 礼品信息：配置型数据，变化少
- 预估数据量：每孩每天 ~10-20 条记录

### 3.2 消息推送
- 每日打卡提醒（定时通知）
- 积分变动通知（可选）
- 成就达成通知

### 3.3 离线支持
- 打卡操作应支持离线
- 有后端时需考虑离线同步策略
